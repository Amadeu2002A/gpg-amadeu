GPG — guía rápida con comandos

Página práctica que explica los comandos GPG (GNU Privacy Guard) más habituales, con ejemplos listos para copiar y ejecutar en la terminal.

¿Qué es GPG?

GPG es una implementación libre de OpenPGP. Permite generar pares de claves (pública/privada), cifrar mensajes o archivos, firmarlos digitalmente y verificar firmas. Es una herramienta esencial para seguridad y privacidad en comunicación y almacenamiento.

Flujo básico (resumen)

  • 1. Generar par de claves (privada + pública).
  • 2. Compartir tu clave pública con quien quiera enviarte datos cifrados o verificar firmas.
  • 3. Cifrar para la clave pública del destinatario.
  • 4. Firmar con tu clave privada para garantizar integridad y origen.

Comandos útiles

Generar un par de claves interactivo

gpg --full-generate-key
# seguir las opciones en pantalla (tipo de clave, tamaño, caducidad, nombre y email)
Usa --batch para automatizar en scripts.

Listar claves (local)

gpg --list-keys         # claves públicas
gpg --list-secret-keys  # claves privadas

Exportar / Importar

# exportar clave pública
gpg --armor --export tu_email@example.com > mi_clave_publica.asc

# importar clave pública de otra persona
gpg --import clave_de_otra_persona.asc

Cifrar y descifrar archivos

# cifrar para destinatario (usa su clave pública)
gpg --output secreto.gpg --encrypt --recipient 'destinatario@example.com' secreto.txt

# descifrar (el receptor usa su clave privada)
gpg --output secreto.txt --decrypt secreto.gpg

Firmar y verificar

# firma clara (texto legible + firma)
gpg --clearsign mensaje.txt

# firma binaria y archivo separado
gpg --output mensaje.sig --sign mensaje.txt

# verificar firma
gpg --verify mensaje.sig mensaje.txt

Exportar clave privada (con cuidado)

# exportar privada (proteger el archivo resultante)
gpg --armor --export-secret-keys tu_email@example.com > mi_clave_privada.asc
# NO compartas este archivo sin extremo cifrado y control

Revocar clave

# generar un certificado de revocación (recomendado al crear la clave)
gpg --output revocacion.asc --gen-revoke tu_email@example.com

# publicar o guardar el certificado para usarlo si la clave se ve comprometida

Consejo: nunca compartas tu clave privada. Mantén frases de paso fuertes y considera usar un agente (ssh- or gpg-agent) para no introducir la passphrase a mano todo el tiempo.
